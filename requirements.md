1. Purpose
Improve the Cardiff University Open Day website so it is easier to use, visually appealing, mobile‑friendly, and accessible.

2. Functional Requirements
Load Open Day data from the JSON file.
Display all subjects with name, description, and image.
Allow users to filter subjects using a dropdown.
Show event details when a subject is clicked.
Sort events by time.
Collapse other subjects when one is opened.

3. UI / UX Requirements
Add Cardiff University branding (red banner).
Use a modern card layout for subjects.
Make the filter bar sticky when scrolling.
Ensure the design works well on mobile and desktop.

4. Accessibility Requirements
Include a skip link that jumps to the main heading.
Make subject cards keyboard‑accessible (tab + Enter).
Use ARIA attributes for expandable sections.
Maintain good colour contrast.

5. Non‑Functional Requirements
Must load quickly on GitHub Pages.
Must be responsive across all screen sizes.
Code should be easy to maintain and update.

6. Assumptions
JSON structure is fixed.
No backend or authentication needed.
Only front‑end technologies (Vite, TypeScript, Tailwind) are used.
